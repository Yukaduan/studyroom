# 专注自习室 @yuka —— 中式教育下的拟真自律学习辅助终端

`专注自习室 @yuka` 是一款专为学生及备考人群设计的单网页互动式自习辅助应用。它巧妙融合了经典“高中教室后门”这一拟真场景，并加入班主任突击检查、纪律委员警报等趣味自律制约机制。

该项目采用单 HTML 文件架构，基于 React 和 Tailwind CSS 编写，开箱即用，无需繁琐部署。

——2026.6.6 小小黑客松活动现场制作。

---

## 📌 核心功能与设计亮点

### 1. 班主任拟真巡视机制（防切屏自律）
* **随机突击抽查**：自习开始后，系统将处于全屏监督状态。在专注周期内，系统会模拟班主任从走廊路过或推门视察的突击抽查。
* **分阶段概率调整**：巡视概率具备智能曲线，前 5 分钟属于沉浸期（概率较低），临近结束阶段盯防最为严格。
* **切屏/偏离焦点判定**：视察期间，若用户退出全屏、切页或隐藏标签，将被直接判定为“开小差”并记名扣分。

### 2. 纪律委员通风报信
* **主动召唤巡查**：在自习期间，用户可按下 **`Space`（空格键）** 或点击界面中的“通风报信”按钮。
* **拟真交互**：触发后，纪律委员将在通报板低声发出预告警报并响起警示音，随后立即进入班主任突击检查环节，提高自习过程中的互动性和可玩性。

### 3. 多平台响应式与全屏机制兼容
* **门中提示对齐**：所有突击检查的警示条、脚步渐进进度条，均统一对齐并融合于“教室后门”的正中央，在大小屏幕上都有稳定的视觉比例。
* **手机端模拟全屏**：针对部分不支持原生 Fullscreen API 的移动浏览器（如 iOS Safari），系统内置了“模拟全屏”逻辑，不仅能将网页画面最大化，且能借助 `visibilitychange` 事件同样保证切屏惩罚的正常运作。

### 4. 房间高度个性化配置
* 支持对自习室名称、当前自习科目、冲刺考试目标及倒计时日期进行配置。
* 支持自由编写黑板报标语/座右铭，且所有配置均通过 `localStorage` 自动实现本地持久化，刷新不丢失。

### 5. 纯算法声效模拟
* 使用 Web Audio API，无需加载任何外部音频文件。系统会在后台通过正弦波和三角波物理合成脚步声、警示音和提示音，确保页面在弱网环境下也能获得即时声音反馈。

## 🛠️ 技术选型

* **核心框架**：React 18 (通过 CDN 在线加载，无需构建步骤)
* **编译器**：Babel Standalone (实时编译 JSX)
* **样式库**：Tailwind CSS (响应式布局与极简设计)
* **音频驱动**：Web Audio API (动态音频合成)
* **持久化**：Web Storage API (本地保存设置和专注档案)

## 🚀 快速开始

### 运行环境
由于系统需要通过 CDN 引入 React、Tailwind CSS 等脚本资源，**运行时需要保持设备处于联网状态**。

### 启动步骤
1. 将 `index.html` 文件下载到本地任意目录下。
2. 双击 `index.html`，或者在浏览器中选择“打开文件”。
3. 首次使用时，建议先点击右上角的 **⚙️ 房间配置**，输入您当前的复习计划和高考/目标考试日期。
4. 点击 **“开始进入全屏自习”** 即可开始体验。

## ⌨️ 快捷操作指引

* **`Space`（空格键）**：在开始自习后，可按空格键让纪律委员呼唤班主任进行突击抽查。
* **双击或切屏**：在突击抽查期间如果进行以上动作，将触发违纪通报。
* **白名单外链**：当需要查询学习资料时，可点击右侧白名单网站。系统将自动发放 3 分钟假条并允许退出全屏，超时未归则视作开小差。

---

## ⚠️ 注意事项

1. **音频播放权限**：由于现代浏览器（尤其是移动端 Safari 和 Chrome）有严格的安全策略，背景声和警示音可能在用户首次点击屏幕后才会正常激活。
2. **全屏机制受限说明**：
   * 原生全屏只能通过用户主动点击触发（如点击“开始自习”按钮），无法由代码无感强行开启。
   * 在部分低版本手机浏览器或第三方应用内置浏览器中，可能会由于宿主权限限制导致无法原生全屏。遇到此类情况，系统将无缝降级采用**网页级模拟全屏**（依然受到切屏检测约束）。
